# OAuth 연동 흐름

GTDonation-Chzzk는 플레이어가 서버에 접속할 때 자동으로 치지직 OAuth 인증을 처리합니다.

---

## 인증 흐름

```
플레이어 접속
    ↓
DB에 UUID 존재 여부 확인
    ↓ (없음)                           ↓ (있음)
채팅창에 로그인 링크 전송          토큰 만료 여부 확인 (29일)
    ↓                                  ↓ (만료)        ↓ (유효)
플레이어가 링크 클릭              재인증 링크 전송   세션 자동 연결
    ↓
치지직 로그인 & 권한 동의
    ↓
서버 콜백 수신 (HTTP 서버)
    ↓
토큰 저장 (SQLite) → 세션 연결
```

---

## 1. 최초 로그인

플레이어가 서버에 처음 접속하면 채팅창에 링크가 나타납니다.

```
[CHZZK] 로그인 링크를 클릭하세요: 여기를 클릭
```

링크를 클릭하면 아래 형식의 치지직 인증 페이지로 이동합니다.

```
https://chzzk.naver.com/account-interlock
  ?response_type=code
  &clientId={CLIENT_ID}
  &redirectUri={http(s)://displayHost:port/callbackPath}
  &state={플레이어 UUID}
```

치지직 계정으로 로그인하고 권한을 동의하면 서버의 Redirect URI로 콜백이 전달됩니다.

---

## 2. 콜백 수신

GTDonation-Chzzk는 플러그인 로드 시 내장 HTTP 서버를 `port`에 띄웁니다. 콜백을 수신하면 액세스 토큰과 리프레시 토큰을 발급받아 SQLite DB에 저장합니다.

인증 완료 시 플레이어에게 메시지가 전송됩니다.

```
[CHZZK] 인증이 완료되었습니다.
```

---

## 3. 토큰 자동 갱신

이후 접속부터는 DB에 저장된 토큰으로 자동 연결됩니다. 마지막 갱신으로부터 **29일**이 경과하면 재인증을 요청합니다.

```
[CHZZK] 인증이 만료되었습니다. 다시 인증하려면 클릭하세요.
```

---

## 4. 연동 해제

플레이어가 직접 연동을 해제할 수 있습니다.

```
/chzzk unlink
```

세션이 종료되고 DB에서 토큰이 삭제됩니다.
